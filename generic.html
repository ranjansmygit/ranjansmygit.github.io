Principal Security Engineer

As a Principal Security Engineer, I spearheaded the integration of security throughout the DevOps lifecycle for banking mobile applications, emphasizing automation and continuous security within the CI/CD pipeline. Using SonarQube and Fortify for static code analysis, OWASP ZAP for dynamic application security testing, and Aqua Security for container vulnerability scanning, I developed a secure Software Development Lifecycle (SDLC) framework. This comprehensive automation reduced human error and expedited secure application releases, significantly reducing security risks across multiple banking regions.

Projects

Designed and implemented robust automated security solutions for vulnerability management and brute-force attack detection. For patch management, I developed an Automated Vulnerability Prioritization System using pandas and scikit-learn, enabling real-time risk assessment by aggregating data from vulnerability scans and generating custom risk scores based on asset criticality and exploitability. By leveraging Gradient Boosting Classifier and K-means clustering for accurate vulnerability ranking and grouping, coupled with matplotlib and Seaborn visualizations, I streamlined reporting and enabled security teams to focus on high-risk issues, integrating this solution into Splunk and Jira for continuous monitoring and efficient remediation. In parallel, I developed a Login Pattern Analysis System for detecting brute-force attacks on banking applications. This involved using DBSCAN clustering and Random Forest within a time-series framework to analyze login patterns and flag abnormal login behavior in real time, effectively identifying brute-force attempts. The solution was fully integrated into the Splunk SIEM and visualized via matplotlib dashboards, providing the Security Operations Center (SOC) with immediate, high-confidence alerts, thereby reducing unauthorized access risks and enhancing incident response efficiency.
